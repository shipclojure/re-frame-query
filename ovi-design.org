#+title: Re Frame Query design

* Terms
- =:stale-time-ms= - time that a query data is valid until a refetch is attempted
- =:cache-time-ms= - time data is kept in cache (re-frame db) from the last
  active subscription of that data, until it is cleared

* Features

** Tag invalidation with support for dedicated tags
This means that if I have a query =todo/list= and a by id tag =todo/get= with
aditional ID, I am able to define cache invalidation based on dynamic tag
generation defined at mutation definition


#+begin_src clojure
  (rfq/register-query :todo/get
                      {:query-fn
                       (fn [{:keys [id on-success on-error]}]
                         {:http-xhrio {:method :post
                                       :uri (str "/api/todo" id)
                                       ...
                                       }})
                       :}

#+end_src


#+begin_src clojure
  (rfq/register-mutation :todo/delete
                         {:mutation-fn (fn [{:keys [id]}] {...})
                          :invalidates-tags #{[:todo/list]}
                          ;; Or receive a function with the query parameters
                          :invalidates-tags (fn [{:keys [id]}] #{[:todo/list] [:todo/get id]}

#+end_src
